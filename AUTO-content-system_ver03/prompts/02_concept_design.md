# Phase 2: コンセプト設計

## 概要

リサーチ結果をもとに**二層構造のコンセプト**を設計する。
売れるコンセプトには「実利」と「世界観」の両方が必要。

**目標**: コアコンセプト + 抽象的コンセプト + コンセプト文 + 仮想ストーリーを確定する

**手法**: コアコンセプト/抽象的コンセプト設計 + 4要素フレーム + コンセプト5要素 + コンセプト構築プロセス

---

- `output/00a_self_analysis.md`（自己分析レポート）※あれば
- `output/01_research.md`（競合リサーチレポート）
- ヒアリング情報

## コンセプトの二層構造

### 層1: コアコンセプト（実利的価値）

顧客への**夢の約束**。商品を買うことで得られる具体的な成果。

```
公式: [ターゲット]が[期間]で[具体的ゴール]を達成する[手段]

例:
- 「Threadsを使って月50万稼ぐ方法」
- 「出した瞬間に売れるnoteのテンプレート」
- 「フォロワー500人でも月5万稼ぐアフィ術」
```

### 層2: 抽象的コンセプト（世界観・ブランド）

商品の**独自の世界観**。記憶に残り、ファンを生む要素。

```
例:
- 「秘伝書」← 禁断のノウハウ感
- 「裏バイブル」← 秘密の知識感
- 「設計図」← 体系的・再現可能感
- 「処方箋」← 悩み解決感
- 「魔法のテンプレ」← 簡単・即効性感
```

**抽象的コンセプトの作り方:**
1. テーマから連想される**メタファー**を10個書き出す
2. その中で「ターゲットの感情に刺さる」ものを選ぶ
3. 商品名・章タイトル・表現に統一的に使う

## 実行手順

### STEP 1: 悩みの深掘り

リサーチで見つけた悩みを**5段階で深掘り**する。

```
表面の悩み: [リサーチで出た悩み]
  ↓ なぜそれが悩みなのか？
第2層: [理由]
  ↓ それが解決しないとどうなる？
第3層: [恐怖・損失]
  ↓ 本当はどうなりたい？
第4層: [根本的な願望]
  ↓ それが叶ったらどんな感情？
第5層: [究極の感情ゴール]
```

**例:**
```
表面: 「noteが売れない」
  ↓
第2層: 「自分の知識に価値がないと感じる」
  ↓
第3層: 「このまま副業が成功しないかも」
  ↓
第4層: 「会社に依存せず自由に生きたい」
  ↓
第5層: 「自分の力で生きている実感がほしい」
```

→ セールスレターではこの**全層**に訴求する

### STEP 2: 価値の転換

自分の経験を「ターゲットにとっての価値」に変換する。

```
■ 変換テンプレート
自分の経験: [自分がやったこと]
  ↓
ターゲットの悩み: [この経験が解決する悩み]
  ↓
提供する価値: [具体的に何を教えられるか]
  ↓
独自の切り口: [他の人には語れない角度]
```

### STEP 3: 4要素の決定

以下の4要素をそれぞれ決定する。

#### 要素1: 誰が（ターゲット）

ターゲットを**3つの条件**で具体的に絞り込む。

**決め方:**
- リサーチで見つけた「競合が拾えていない層」を狙う
- 悩みの深掘りで「最も感情が動く層」を選ぶ

```
ターゲット: [大カテゴリ]
条件1: [すでに持っている前提条件]
条件2: [抱えている具体的な課題]
条件3: [現在の状況・レベル]
```



#### 要素2: どこまで（ゴール）

**ゴールの条件:**
- 数字が入っている
- 達成したかどうか判定可能
- 高すぎない（期間内で達成できる範囲）
- ターゲットが「それ欲しい！」と思える

**良い例 vs 悪い例:**
- △「商品を作る」→ ◎「2,980円の商品を完成させて初売上を出す」
- △「稼げるようになる」→ ◎「月20万円の売上をローンチで達成する」


#### 要素3: どうやって（手段・独自メソッド）

**独自メソッドの作り方（ポジション×切り口×自分の掛算公式）:**
```
ポジション: [自分が語れる立場]
切り口: [他とは違う角度]
自分の強み: [経験・スキル]
  ↓
独自メソッド: [ポジション] × [切り口] = [メソッド名]
```

**例:**
- テンプレートをフル活用 → 「コピペ完結メソッド」
- AIを組み合わせて → 「AI×人力ハイブリッド術」
- 3ステップの簡易フレームワーク → 「3ステップ速攻フレーム」


### STEP 4: コンセプト文の生成

**テンプレート（コンセプト5要素）:**
```
[ターゲット]が
[プロセス/手段]によって
[期間]で[ゴール]する

独自性: [他にない自分だけの要素]
ストーリー: [なぜ自分がこの商品を作ったか]
```

**生成例:**
```
恋愛に自信がない男性  
・好きな人はいる  
・出会いはある  
・でもどうアプローチすればいいか分からない  
という人が、  

心理学ベースの会話テンプレートと  
自然に距離を縮める行動ステップを使うことで  

「いい人止まり」を卒業し  
本命女性とデート関係まで進める恋愛コンテンツ  

独自性: 「考えなくても使える恋愛会話テンプレート＋行動ロードマップ」  
ストーリー: 自分も昔は女性と何を話せばいいか分からず  
チャンスを何度も逃していたが、  
心理学と会話パターンをテンプレ化したことで  
自然に関係を深められるようになった経験から
```

### STEP 5: 仮想ストーリーの構築

コンセプトに**感情的な物語**を紐づける。

```
■ 仮想ストーリーテンプレート

【過去】
自分も[ターゲットと同じ状態]だった。
[具体的なエピソード]

【転機】
[きっかけ]が訪れた。
[発見・気づき]をした。

【変化】
[手段]を実践したことで、
[期間]で[成果]を達成できた。

【使命】
だからこそ、[ターゲット]に
[価値]を届けたい。

→ このストーリーがセールスレターの核になる
```

### STEP 6: コンセプトの検証

以下のチェックリストで検証する。

**必須チェック（全て「はい」でOK）:**
- [ ] ターゲットが「自分のことだ」と感じる具体性があるか
- [ ] ゴールに数字が含まれているか
- [ ] 手段に独自性・新規性があるか
- [ ] 網羅系になっているか
- [ ] 競合と明確に差別化できているか
- [ ] コアコンセプト（実利）が1文で説明可能か
- [ ] 抽象的コンセプト（世界観）が決まっているか
- [ ] 仮想ストーリーに感情的な説得力があるか

**NGチェック:**
- [ ] ゴールが曖昧（「稼げるようになる」等）でないか
- [ ] ターゲットが広すぎ（「誰でも」等）でないか

## コンセプトの型（参考）

| 型 | 説明 | タイトル例 | 適した商品 |
|----|------|-----------|-----------|
| 目標達成型 | N日でXを達成する | 7日で0→1突破プログラム | テスト〜メイン |
| 問題解決型 | Xの悩みを解決する | もう迷わないコンセプト設計術 | メイン |
| 変身型 | AからBに変わる | 凡人を脱出する仕組み化メソッド | メイン〜高額 |
| 秘密暴露型 | 知られていないXを公開 | 月収100万の裏側全公開 | テスト〜メイン |
| テンプレ型 | コピペで完成する | 穴埋めするだけの商品設計テンプレ | テスト |
