# Phase 3: 商品設計

## 概要

コンセプトに基づき、商品の**全体設計**を行う。
ステップ設計だけでなく、オファー・カリキュラム・世界観を一体で設計する。

**目標**: オファー設計 + カリキュラム設計 + 世界観設計 + タイトル設計 + 章構成マップを完成させる
**価格帯**: 49,800円クラスのハイクオリティ商品を前提に設計する

**手法**: オファー設計 + 世界観設計 + カリキュラム設計 + タイトル設計

## PART B: カリキュラム設計

### STEP 1: 大ステップの洗い出し

読者が**スタートからゴール**に到達するまでの大きな工程を3〜5個書き出す。

**洗い出しの基準:**
- **時系列順**で並べる（読者が実践する順番）
- 各ステップが「1つの明確な成果物」に対応
- ステップ間に「飛躍」がない（前のステップの結果が次の入力になる）

```
大ステップ1: [名称]　→ 成果物: [何ができるか]
大ステップ2: [名称]　→ 成果物: [何ができるか]
大ステップ3: [名称]　→ 成果物: [何ができるか]
大ステップ4: [名称]　→ 成果物: [何ができるか]
```

### STEP 2: 小ステップの分解（カリキュラムテーブル）

各大ステップを3〜5個の小ステップに分解する。
**商品設計ワークシートのカリキュラム形式**で整理する。

```
| # | ステップ | 課題（読者の壁） | 解決策 | タイトル | タイプ |
|---|---------|---------------|--------|---------|--------|
| 1-1 | [作業名] | [読者がここで直面する問題] | [具体的な解決法] | [魅力的なタイトル] | 解説/ワーク/テンプレ |
| 1-2 | ... | | | | |
```

**タイプの説明:**
- **解説**: 知識・ノウハウの説明
- **ワーク**: 読者が手を動かす演習
- **テンプレ**: 穴埋め式のテンプレート提供
- **事例**: 具体的なケーススタディ
- **チェックリスト**: 確認・振り返り用

### STEP 3: 各小ステップの詳細メモ

```
■ 小ステップ [番号]: [作業名]

課題: [読者がここで直面する壁]
解決策: [具体的に何をすれば解決するか]

何をするか:
（具体的な作業内容を2〜3文で）

なぜやるか:
（この作業の目的・やらないとどうなるか）

コツ / 注意点:
（実践時のポイントを1〜2つ）

成果物:
（この作業が終わると何ができているか）

使うツール / リソース:
（テンプレート・ツール・参考資料）
```

### STEP 4: 章構成への変換

```
コンテンツ構成:

【導入】第1章: コンテンツの目的・全体像・学習手順
         ↑ これはステップとは別に必ず設ける

【本編】第2章 = 大ステップ1
        第3章 = 大ステップ2
        第4章 = 大ステップ3
        第5章 = 大ステップ4
        ↑ 大ステップの数だけ章が増える

【まとめ】最終章: 全体の振り返り・行動喚起
           ↑ これもステップとは別に必ず設ける
```

**各章の中身:**
```
第N章: [大ステップ名]
├── 章の導入文（テーマ・全体像・重要性）
├── 小ステップ1の解説
│   ├── 課題の提示（読者の壁）
│   ├── 解決策の全体像
│   ├── 具体的な手順（ステップバイステップ）
│   ├── 実例・ケーススタディ
│   └── ワーク / テンプレート
├── 小ステップ2の解説
│   └── （同上）
├── 小ステップ3の解説
│   └── （同上）
└── 章のまとめ + 次章への橋渡し
```

## PART D: タイトル設計

### 大前提：noteが売れない3大地雷

タイトル設計に入る前に、以下の3つの地雷を絶対に踏まないこと。

```
【地雷①】抽象的なタイトル
❌「成功する方法」「稼ぐコツ」「人生を変える秘訣」
→ 「何の成功？」「いくら稼ぐ？」「誰の人生？」と疑問を持たせたら負け
→ 読者は抽象的な言葉に一切興味がない。「具体的な結果」に興味がある

【地雷②】読者が欲しい「未来」を見せていない
❌「私の経験をまとめました」「知識をシェアします」
→ 読者はあなたの経験や知識に1ミリも興味がない
→ 読者が求めているのは「この記事を読んだ後の自分の未来（理想の未来）」だけ
→ 「知識をシェア」ではなく「月収を2倍にした思考法」と書く

【地雷③】ターゲット不在の投げ売り
❌「誰でもできる」「全ての人へ」「誰にでも効果あり」
→ 「誰でも」＝「誰でもない」「全ての人」＝「誰一人いない」
→ ターゲットを絞れば絞るほど、その人に「刺さる」コンテンツになる
```

### 爆売れタイトルの方程式

**メインタイトル（何が手に入るかを示す）:**
```
[誰が] + [どれくらいの時間で] + [どんな結果を得られる] + [具体的な方法]
```

**サブタイトル（なぜそれが実現できるかを補足）:**
```
〜[圧倒的な価値・驚き]〜
```

**完全版方程式:**
```
[誰が] + [どれくらいの時間で] + [どんな結果] + [具体的な方法]
〜[圧倒的な価値]〜
```

**具体例:**
```
✅ 会社員が週末2時間で月10万稼ぐメルカリ仕入れ術
  〜在庫0でも利益率200%の転売術を完全公開〜

✅ 主婦がスキマ時間で月3万稼ぐnote販売テンプレ
  〜たった3記事で半年間の不労所得を作る方法〜

✅ note初心者が1日30分で月5万稼げる禁断の文章術
  〜セールスライティング一切不要の新メソッド〜
```

**ポイント:** メインタイトルで「何が手に入るか」、サブタイトルで「なぜそれが実現できるか」を示す。この2層構造が「あ、これ買わなきゃ」と思わせる秘密。

### 圧倒的価値の作り方：5つの鉄則

サブタイトルで「驚き」を作るための5鉄則。人は「驚き」にしかお金を払わない。
全部を詰め込む必要はないが、どれか1つでも強烈なものがあれば売れる。複数組み合わせると購買意欲はさらに跳ね上がる。

```
【鉄則①】競合と真逆を行け
→ 界隈の「常識」を見つけ、その真逆を打ち出す
→ 例: みんな「在庫を抱えろ」→ あなたは「在庫0」
→ 例: みんな「毎日3投稿」→ あなたは「週1投稿」
→ 例: みんな「セールスが重要」→ あなたは「セールス不要」

【鉄則②】常識を破壊する数字を出せ
→ 一般的な数値を見つけ、それを圧倒的に超える数字を出す
→ 例: 普通の利益率30% → あなたは「200%」
→ 例: 普通は毎月更新 → あなたは「半年放置でOK」
→ ※嘘はダメ。実績に基づいた検証済みの数字であることが絶対条件

【鉄則③】時短で結果を約束しろ
→ 普通の期間を大幅に短縮した成果を提示する
→ 例: 普通は3ヶ月 → あなたは「7日で達成」
→ 例: 普通は毎日3時間 → あなたは「1日10分」
→ 現代人の最大の悩みは「時間がない」。時短は最強の価値提案
→ ※ありえない時短は逆効果。「現実的な時短」を提示すること

【鉄則④】手間を最小限に見せろ
→ 複雑な工程を極限まで簡略化した方法を提示する
→ 例: 普通は100記事 → あなたは「3記事だけ」
→ 例: 普通は専門スキル必須 → あなたは「コピペだけ」
→ 「簡単」より「3ステップ」の方が説得力がある。具体的な数字で示す

【鉄則⑤】リスクをゼロにしろ
→ 普通にある初期投資や失敗リスクを消し去る
→ 例: 普通は元手10万円 → あなたは「0円スタート」
→ 例: 普通は専門知識必須 → あなたは「未経験でOK」
→ 人が行動を躊躇する最大の理由は「リスク」。これを取り除く
```

**鉄則の組み合わせ例:**
```
✅ 鉄則④⑤: 「未経験でも0円スタート！たった3記事で月10万円を稼ぐ方法」
✅ 鉄則①③: 「毎日投稿なしで1日10分！放置で月3万円の不労所得を作る方法」
```

### タイトルワード一覧

**方法名に使うワード（目標達成型向け）:**
```
・マニュアル        ・戦略
・ロードマップ      ・テンプレート
・メソッド          ・設計図
・処方箋            ・攻略法
・実践ガイド        ・プログラム
```

**NGワード（抽象的・網羅系ワード）:**
```
❌ 完全攻略  ❌ バイブル  ❌ 教科書  ❌ 大全  ❌ 講座
❌ 成功する方法  ❌ 稼ぐコツ  ❌ 人生を変える秘訣
❌ 誰でもできる  ❌ 全ての人へ  ❌ 知識をシェア
```

### タイトル生成の手順

1. **地雷チェック**: 3大地雷（抽象的・未来なし・ターゲット不在）に該当しないか確認
2. **方程式でメインタイトル候補を5つ生成**: `[誰が] + [時間] + [結果] + [方法]`
3. **5鉄則でサブタイトル候補を5つ生成**: 競合リサーチから「真逆」「常識破壊」「時短」「手間最小」「リスクゼロ」のどれが最も強いか判断し、サブタイトルに組み込む
4. **「驚き」チェック**: 読者が「えっ！そんなことできるの！？」と思えるかを検証。驚きがなければ書き直す
5. **最終選定**: 「誰に・何を・どうやって」が最も明確で、かつ「驚き」が最も強い組み合わせを選定

### タイトル評価基準

候補を以下の5項目で採点する（各5点、合計25点満点）。

```
| 評価項目 | 基準 |
|---------|------|
| ① 具体性 | 「誰が・何を・どうやって」が明確か？数字が入っているか？ |
| ② 理想の未来 | 読者が手に入れられる「未来の姿」が見えるか？ |
| ③ ターゲット明確性 | 「これは自分のことだ」と思える人が明確か？ |
| ④ 驚き・意外性 | 「えっ！？」という驚きがあるか？常識を覆す要素があるか？ |
| ⑤ 価値の圧倒感 | サブタイトルで「なぜ実現できるか」の圧倒的な理由が示されているか？ |
```

**合格ライン: 20点以上。20点未満の候補は採用しない。**
