# 有料コンテンツ作成 — マスターオーケストレーター
## 概要

本プロンプトは「売れる有料noteを完成させる」ための全6フェーズを
連鎖的に実行する統括プロンプトである。

**統合知識ベース**
- 自己分析・コンセプト設計・ファン化
- コンテンツ融合法・連載ストーリー型運用
- コンセプト構築プロセス
- オファー設計・世界観設計・カリキュラム設計
- たこすけ式構成法・13要素下書き構造

---

## 実行前ヒアリング

以下の情報をユーザーから収集する。不明な項目はPhase 0〜1で補完する。

```
■ 基本情報
1. テーマ・ジャンル: （例：note販売、Threads運用、ダイエット、恋愛、占い）
2. ターゲット: （例：note初心者で商品が作れない人）
3. あなたの実績・経験: （例：noteで月30万、3か月で-10kg）
4. 教えるノウハウの要点: （箇条書きで10個）
5. 想定価格帯: （テスト商品: 1980〜4,980円 / メイン商品: 9,800〜49,800円）
6. 想定ボリューム: （例：25,000字）
7. 販売プラットフォーム: （note / Brain）
8. 一人称: （僕 / 私 / わたし / ワイ）
9. 文体: （カジュアル / 解説型 / 体験談ベース / リーダー型 / ちょっと上の先輩）

■ 深掘り情報（任意・Phase 0で補完可能）
10. あなたの過去の挫折・失敗: （共感ストーリー用）
11. ターゲットの最大の恐怖: （セールスレター用）
12. 競合で気になる商品: （リサーチの出発点）
13. 独自メソッド名の候補: （ブランディング用）
14. 商品の世界観イメージ: （秘伝型 / シンプル型 / 図解型）
```

## 実行フロー

以下のタスクを順番に実行する。
保存先パス: 現在のワーキングフォルダ（`output/` または `output2/`, `output3/`...）

**⚠️ 重要: 各プロンプト内の `output/` は全て「現在のワーキングフォルダ」を意味する。**
実際のパスは CLAUDE.md のフォルダ自動採番ルールで決定されたフォルダに読み替えること。

## 実行ルール

1. 各タスクは必ず前タスクの保存ファイルを読み込んでから実行すること
2. 各タスクの結果は指定フォルダに指定ファイル名で保存すること
3. フェーズ間で矛盾が生じた場合はコンセプト（タスク2）を優先すること
4. 全タスク完了後、最終サマリーを `00_summary.md` として同フォルダに保存すること
5. **コンセプトの一貫性**: 全フェーズを通じてコアコンセプトと抽象的コンセプトがブレないこと
6. **ターゲットの解像度**: 具体的な1人の人物像が全フェーズで統一されていること

## 商品タイプ判定

ヒアリング情報から以下の商品タイプを自動判定する。

| タイプ | 価格帯 | ボリューム | 用途 |
|--------|--------|-----------|---------|------|
| テスト商品 | 1980〜4,980円 | 5,000〜15,000字  | 市場テスト・実績作り |
| メイン商品 | 9,800〜49,800円 | 20,000〜50,000字  | 収益の柱 |
| 高額商品 | 50,000円〜 | 50,000字+  | バックエンド |


**テスト商品の場合**: Phase 0をスキップ可、Phase 3を簡略化
**メイン商品の場合**: 全Phase実行、世界観設計を重点的に
**高額商品の場合**: 全Phase + サポート設計・特典設計を強化

## 品質基準

| 項目 | 基準 |
|------|------|
| コンセプトの二層構造 | コアコンセプト（実利的価値）+ 抽象的コンセプト（世界観）が両立 |
| ターゲットの具体性 | 「自分のことだ」と思える具体的な1人が描けている |
| ステップの実行可能性 | 各ステップに数字・分量・時間目安・成果物が含まれている |
| 有料部分のたこすけ式構成法 | Problem→Amplify→Solution→Testimony→Offerの流れが各章に存在 |
| ノウハウの実用性 | 読者が読んだ直後に行動に移せるレベルの具体性 |
| セールスレターの訴求力 | 好奇心→欲望→共感→絶望→否定→希望→信頼→恐怖→興奮→納得→行動の感情曲線がある |
| 世界観の一貫性 | ワーディング・ビジュアルイメージが全文で統一されている |
| 文体の一貫性 | 全文を通じて同一の文体・トーンが維持されている |
| スマホ最適化 | 1段落4行以内、適切な改行・装飾がある |
| 独自性 | コンテンツ融合法により3つ以上のソースを融合した独自の切り口がある |
