# Phase 5: 無料部分（セールスレター）作成

## 概要

読者はこの**無料部分**を見て購入を決断する。
**感情設計フロー × たこすけ式構成法 × 17セクション × 心理トリガー**の四重構造で最強のレターを作る。

**目標**: 5,000字以上の高CVRセールスレターを完成させる

**手法**: 17セクション感情設計 + たこすけ式構成法 + 3つの刷り込み + ザイオンス効果 + 感情曲線設計

## 最重要原則

**読者は論理ではなく感情で購入を決定し、後から論理で正当化する。**
この感情の流れを壊さないことが最重要原則。

---

## 基本ルール（9箇条）

1. **必ず17セクションの構成順序に従うこと。** 順序の入れ替えは原則禁止。ただし商材特性上不要なセクションはスキップ可。
2. **各セクションには明確な「感情目標」がある。** その感情目標を達成する文章を書くこと。
3. **一人称は商品設計書のワーディングに従う。** 読者への呼びかけは「あなた」で統一。
4. **具体性が命。** 抽象的な表現（「すごい」「ヤバい」）は必ず具体的な描写・数字・シーンに置き換える。
5. **1文は60文字以内を目安。** 短文でリズムを作り、読者を止めない。
6. **改行を多用し、スマホでの可読性を最優先する。**
7. **セールス臭を消す。** 「買ってください」ではなく「知りたくないですか？」のように、読者の好奇心と自発的意思を引き出す表現を使う。
8. **購入者の声・実績は可能な限り挿入する。** 社会的証明は最低2箇所に配置。
9. **CTAは1つに絞る。** 複数の行動を求めない。

---

## トーン＆ボイス

- 友人に語りかけるような親しみのある口調
- ただし馴れ馴れしすぎない（敬語ベース＋時折タメ口を混ぜる）
- 感情が高まるシーンでは短文・体言止めを多用
- 冷静なシーン（ロジックブリッジ・価格提示）では丁寧語に戻す
- 全体を通じて「読者の味方である」というスタンスを崩さない
- 商品設計書のワーディング設計に従う

## 心理トリガーの基盤

### 感情設計の全体フロー

```
好奇心 → 欲望 → 共感 → 絶望 → 否定 → 希望 → 信頼 → 恐怖 → 興奮 → 納得 → 行動
```

この感情フローを17セクションで具体化する。

### 感情のジェットコースター設計

```
高  ┃  ★希望          ★希望
    ┃ ／＼           ／＼
感  ┃／    ＼       ／    ＼
情  ┃        ＼   ／        ＼→★購入
    ┃          ＼／
低  ┃          ★恐怖
    ┗━━━━━━━━━━━━━━━━━━━━
     冒頭    中盤     終盤
```

重要なのは「谷」を作ること。ベネフィットだけを並べても感情は動かない。
恐怖・絶望・否定のパートが「谷」を作り、その直後の希望が際立つ。

### 3つの刷り込み

セールスレターの各セクションで以下の3つを読者の潜在意識に刷り込む。

```
① 教育者認知: 「この人はこの分野の専門家だ」
   → SEC05（権威づけ数値）、SEC08（既存解決策の否定）、SEC10（ロジックブリッジ）で構築

② 結果出せる認知: 「この人のノウハウなら結果が出る」
   → SEC03（ベネフィット）、SEC09（アフターストーリー）、SEC11（社会的証明）で構築

③ 仲間認知: 「この人は自分の味方だ、理解してくれている」
   → SEC06（痛みの代弁）、SEC07（ビフォーストーリー）、SEC14（販売動機の告白）で構築
```

### ザイオンス効果（リフレインの法則）

セールスレター内で**コンセプトのキーメッセージを最低7回**繰り返す。
表現を変えながら、同じ核心メッセージを刷り込む。

```
1回目: SEC02（サブキャッチ）で初出
2回目: SEC03（ベネフィット）で欲望として再提示
3回目: SEC09（アフターストーリー）で変化の証拠として再提示
4回目: SEC10（ロジックブリッジ）で論理的根拠として再提示
5回目: SEC13（希望訴求）で理想の未来として再提示
6回目: SEC15（商品内容）で具体的内容として再提示
7回目: SEC17（追伸+CTA）で最終メッセージとして再提示
```

冒頭のキャッチコピーと同じフレーズを追伸で再利用すると効果的。

## セールスレターの全体構造

### 感情フロー × たこすけ式構成法 × 17セクション マッピング

```
■ 欲望パート（好奇心→欲望）
  SEC01: キャッチコピー（フック）          ← 好奇心の点火
  SEC02: サブキャッチ（内容の方向づけ）    ← 自分ごと化
  SEC03: ベネフィットリスト（欲望の増幅）  ← T: 成功の証拠
  SEC04: 裏ベネフィット（注意喚起型の欲望刺激）
  SEC05: 権威づけ数値（論理の補強）        ← T: 権威

■ 共感・問題パート（共感→絶望→否定）= P
  SEC06: ターゲットの痛みの代弁           ← P: 問題提起 → ③仲間認知
  SEC07: 販売者のビフォーストーリー       ← P: 同一化  → ③仲間認知
  SEC08: 既存解決策の否定                 ← P: 根本原因 → ①教育者認知

■ 希望・信頼パート（希望→信頼）= S + T
  SEC09: アフターストーリー（変化の証明）  ← S: 解決策  → ②結果出せる認知
  SEC10: ロジックブリッジ                 ← S: 論理根拠 → ①教育者認知
  SEC11: 社会的証明（購入者の声・第1波）   ← T: 証拠   → ②結果出せる認知

■ 感情の振り幅パート（恐怖→興奮）= A
  SEC12: 恐怖訴求（何もしない未来）       ← A: 問題の拡大
  SEC13: 希望訴求（変わった未来）          ← 恐怖との対比で感情振幅最大化

■ オファーパート（納得→行動）= O
  SEC14: 販売動機の告白                   ← O: 信頼構築 → ③仲間認知
  SEC15: 商品内容の公開                   ← O: 商品紹介
  SEC16: 価格提示とアンカリング           ← O: 価格の納得
  SEC17: 追伸＋CTA＋購入障壁の除去       ← O: 最終CTA
```

## 各セクションの生成テンプレート

### SEC01: キャッチコピー（フック）

**感情目標:** 好奇心の点火。スクロールを止める。
**配置:** レター最上部（ファーストビュー）

| 要素 | 説明 |
|------|------|
| 機能 | 3秒以内に「何これ？」と思わせる |
| 文字数 | 15文字以内が理想。最大25文字 |
| 技法 | 不可逆性（「一度知ったら戻れない」）、禁忌感（「本当は教えたくない」）、矛盾（「頑張らないほど結果が出る」） |
| 禁止 | 商品名を入れない。説明しない。売り込まない |
| 判定基準 | 「続きを読まないと気になって仕方ない」と感じるか |

```markdown
{キャッチコピー — 15文字以内}
```

**テンプレート例:**
- 「一度◯◯したら戻れない」
- 「◯◯を知ってしまった人の末路」
- 「なぜ◯◯するだけで△△になるのか」
- 「【閲覧注意】◯◯を◯◯しただけで…」


**生成のコツ:**
- コンセプト設計書の抽象的コンセプトから世界観を抽出
- ターゲットの「最も深い願望」を刺激するワードを選ぶ
- 答えをレター内に求めさせる「情報ギャップ」を作る


---

### SEC02: サブキャッチ（内容の方向づけ）

**感情目標:** 好奇心を「自分ごと」に変換する → **キーメッセージ1回目**
**配置:** キャッチコピー直下

| 要素 | 説明 |
|------|------|
| 機能 | このレターが何について書かれているのかを1文で伝える |
| 文字数 | 30〜50文字 |
| 技法 | 「このnoteでは◯◯に特化した【△△する方法】を公開します」の型 |
| 判定基準 | 読者が「これは私のための文章だ」と認識できるか |

```markdown
このnoteでは{ターゲット}に特化した
**【{期間}で{ゴール}する方法】** を公開します。
```

---

### SEC03: ベネフィットリスト（欲望の増幅）

**感情目標:** 「手に入れたい」という欲望を具体的に膨らませる → **キーメッセージ2回目**
**配置:** サブキャッチ直下

| 要素 | 説明 |
|------|------|
| 機能 | 商品を手に入れた後の「変化」を5〜7個、箇条書きで提示 |
| 技法 | 結果→感情→他者からの評価の3層で書く |
| 順序 | 最も共感される悩みの解消 → 意外な副次効果 → 人間関係の変化 |
| 禁止 | 機能の説明（「全10章」「PDF形式」）は絶対に入れない |
| 判定基準 | 各項目を読んで「それ欲しい」と思えるか |

```markdown
このnoteを読むと…

・{具体的な変化1} → {それによって得られる感情/状態}
・{具体的な変化2} → {感情/状態}
・{具体的な変化3} → {感情/状態}
・{具体的な変化4} → {感情/状態}
・{具体的な変化5} → {感情/状態}
```

**生成のコツ:**
- コンセプト設計書のゴール＋サブベネフィットから展開
- リサーチの「読者が求めていること TOP5」を活用
- 悩みの深掘り第4〜5層の「本当の願望」を使う

---

### SEC04: 裏ベネフィット（注意喚起型の欲望刺激）

**感情目標:** 「良すぎる結果」への好奇心をさらに煽る
**配置:** ベネフィットリスト直下

| 要素 | 説明 |
|------|------|
| 機能 | 「逆にこんな現象に注意」と注意喚起の体裁で、実質的には魅力を提示する |
| 技法 | ネガティブフレーム（注意書き・警告の形式）で書くことで、読者の防御を下げつつ欲望を刺激 |
| 個数 | 3〜5個 |
| 判定基準 | 読者が「こんな注意なら大歓迎」と思えるか |

```markdown
逆にこんな現象に注意⚠️

・{一見ネガティブだが実は嬉しい結果1}
・{他者への影響が出るレベルの変化2}
・{周囲が驚くレベルの変化3}
```

---

### SEC05: 権威づけ数値（論理の補強）

**感情目標:** 「本当にそんなことあるの？」という疑念を数字で抑える → **①教育者認知**
**配置:** ベネフィット群の直後

| 要素 | 説明 |
|------|------|
| 機能 | 効果の大きさを数値で示し、同時に希少性を演出する |
| 技法 | 「◯◯は通常のX倍の効果がある」＋「しかし成功している人は全体のたったY%」 |
| 注意 | 出典なき数値は「体感値」「個人の感覚」であることを暗示する程度に留める。嘘は書かない |
| 判定基準 | 「マジで？」と前のめりになるか |

```markdown
{テーマ}に関する興味深いデータがあります。

{効果の数値データ — 自分の実績・クライアント実績から}

でも、この方法を正しく実践できている人は
全体の**たった{○}%** です。

{なぜ少ないのかの理由を1文で}
```

### SEC06: ターゲットの痛みの代弁（共感パート）

**感情目標:** 「この人は私の気持ちをわかっている」という深い共感 → **③仲間認知**
**配置:** レターの中盤前半（ストーリー導入部）

| 要素 | 説明 |
|------|------|
| 機能 | ターゲットが今まさに抱えている悩み・苦しみを、ターゲットの内面の言葉で再現する |
| 技法 | 内面の独白形式。「」で囲んだ自問自答を3〜5個並べる |
| 深度 | 表面の悩み → 本音の悩み → 誰にも言えない悩みの3層構造 |
| 禁止 | 上から目線の分析。「あなたはきっと◯◯ですよね」は避け、「こういう経験ありませんか？」に変換 |
| 判定基準 | 読者が「なんで私の心を読めるの？」と感じるか |

```markdown
こんな経験、ありませんか？

「{Layer1: 表面の悩み — ◯◯がうまくいかない}」

「{Layer2: 本音の悩み — 自分は◯◯なのかもしれない}」

「{Layer3: 秘密の悩み — 誰にも言えないけど、実は◯◯}」

{ターゲットが心の中で思っていること}
{自信のなさ・不安・焦りを言語化}

{最も深い恐怖を1文で — 悩みの深掘り第3〜4層から}
```

**3層の深度モデル:**
```
Layer 1（表面）：「◯◯がうまくいかない」
Layer 2（本音）：「自分は◯◯なのかもしれない」（自己否定）
Layer 3（秘密）：「誰にも言えないけど、実は◯◯」（恥・罪悪感）
```

**生成のコツ:**
- コンセプト設計書のターゲット条件2（課題）から展開
- リサーチの「悩みTOP10」「読者が求めていること TOP5」を活用
- **ターゲットが実際に使っている言葉をそのまま使う**（リサーチから）
- 「あなたはきっと〜」ではなく「こういう経験ありませんか？」で問いかける

### SEC07: 販売者のビフォーストーリー（過去の共有）

**感情目標:** 「この人も同じだったんだ」という同一化 → **③仲間認知**
**配置:** 痛みの代弁の直後

| 要素 | 説明 |
|------|------|
| 機能 | 販売者自身が過去にターゲットと全く同じ悩みを抱えていたことを告白する |
| 技法 | 一人称の物語形式。時系列で具体的なエピソードを描写 |
| 必須要素 | ①具体的な年齢/時期 ②具体的なシーン描写 ③当時の感情の言語化 ④周囲との比較による劣等感 |
| 接続句 | 「これ、完全に昔の私です」で読者の悩みと自分の過去をブリッジする |
| 判定基準 | 読者が「この人は本物だ。信じられる」と思えるか |

```markdown
これ、完全に昔の私です。

{具体的な年齢/時期}の頃、
{具体的なシーン描写 — 五感で想像できる描写}

{当時の感情の言語化 — 内面の独白}

{周囲との比較による劣等感を1〜2文}

{仮想ストーリー（Phase 2で構築済み）のダイジェスト版 — 2〜3文}
```

**生成のコツ:**
- 仮想ストーリー（Phase 2で構築済み）を活用
- 実績は**数字**を必ず入れる
- 自慢ではなく「こうだった自分が変われた」のニュアンス

### SEC08: 既存解決策の否定（情報ギャップの演出）

**感情目標:** 「だからダメだったのか」という腑に落ち＋「じゃあ何が正解なの？」という飢餓感 → **①教育者認知**
**配置:** ビフォーストーリーの中〜後半

| 要素 | 説明 |
|------|------|
| 機能 | ターゲットが既に試した解決策を列挙し、「それでは解決しない理由」を明示する |
| 技法 | 一般論を引用→「そんなことはもうやった」と読者の代わりに否定する |
| 構造 | ①ネットで見つかる一般的アドバイスを3つ引用 ②「そんな精神論ばかり」と切り捨て ③「私が求めてるのはもっと具体的な◯◯」で本商品の方向性を暗示 |
| 判定基準 | 読者が「そう！それが言いたかった！」と膝を叩くか |

```markdown
ネットで調べると出てくるのは
こんなアドバイスばかり。

・「{一般的なアドバイス1}」
・「{一般的なアドバイス2}」
・「{一般的なアドバイス3}」

そんなことはもうやった。

{精神論・根性論への不満を1〜2文}

私が求めてたのは
**もっと具体的な{解決の方向性}** だったんです。
```

### SEC09: アフターストーリー（変化の証明）

**感情目標:** 「自分もこうなれるかも」という希望の着火 → **②結果出せる認知 + キーメッセージ3回目**
**配置:** 否定パートの直後

| 要素 | 説明 |
|------|------|
| 機能 | 販売者が変化した後の具体的な日常を描写する |
| 技法 | 「でも今は違います」で転換し、箇条書きで5〜7個の変化を列挙 |
| 必須要素 | ①感覚の変化（体の反応） ②他者からの評価の変化 ③日常の些細なシーンでの変化 |
| 重要原則 | 大げさな変化と日常的な変化を混ぜる。日常的な変化が信憑性を担保する |
| 判定基準 | 読者が「嘘っぽくない。リアルだ」と感じるか |

```markdown
でも今は違います。

・{日常的で具体的な変化1}
・{感覚の変化2}
・{他者からの評価の変化3}
・{日常の些細なシーンでの変化4}
・{大きな成果の変化5}

{実績を数字入りで3〜5個 — 箇条書き}

**「{理想の状態を1文で}」**
が普通の状態になりました。
```

**信憑性を上げるコツ:**
```
✕「人生が180度変わりました」（抽象的すぎる）
◯「朝起きた瞬間から今日やるべきことが明確で迷いがゼロ」（具体的で日常的）
```

### SEC10: ロジックブリッジ（なぜ変われるのかの論理説明）

**感情目標:** 「なるほど、だから変われるのか」という知的納得 → **①教育者認知 + キーメッセージ4回目**
**配置:** アフターストーリーの直後

| 要素 | 説明 |
|------|------|
| 機能 | 「あなたが悪いのではなく、方法を知らなかっただけ」という論理的根拠を提示 |
| 技法 | 原因の再定義。根本原因→正しい手段→解決への道筋 |
| 注意 | ここは長く書かない。2〜3文で十分。詳細な説明は商品内で行うと暗示する |
| 判定基準 | 読者が「なるほど、方法さえ知れば自分も…」と思えるか |

```markdown
なぜ変われたのか？

結論から言うと、
**「{根本原因を1文で}」**

{ターゲット}が{ゴール}できないのは
あなたのせいじゃない。

**{正しい手段}を知らなかっただけ** です。

{解決への道筋を1〜2文で暗示}
```

### SEC11: 社会的証明（購入者の声・第1波）

**感情目標:** 「自分だけじゃない。みんな成功してる」という安心感 → **②結果出せる認知**
**配置:** ロジックブリッジの直後

| 要素 | 説明 |
|------|------|
| 機能 | 実際の購入者からの成功報告を3〜5件提示 |
| 技法 | スクリーンショット画像（LINEやDM風）が最も効果的。テキストの場合は「」で囲み発言者の属性を添える |
| 必須要素 | ①具体的な変化の描写 ②感情表現 ③実践期間 |

### テクニック01: 感情のジェットコースター設計

セクション全体の感情曲線を意識して「谷」と「山」を交互に配置する。

- 核心メッセージを3回以上繰り返す
- 商品名を含むフレーズを等間隔で配置

### テクニック03: 「画像」の戦略的配置

| 配置タイミング | 画像の役割 |
|-------------|----------|
| ベネフィット後 | 理想の未来をビジュアル化 |
| 購入者の声 | LINEやDMのスクリーンショットで信憑性UP |
| 恐怖訴求後 | 暗いトーンの画像で感情を増幅 |

購入者の声には `[IMAGE: 購入者の声スクリーンショット]` を使う。

### テクニック04: ターゲットの「秘密の言語」を使う

ターゲットが普段使っている言葉遣い（口語・スラング・独自の表現）をリサーチし、レター内で使用する。

```
◯ ターゲットの言語：「もう何やっても無理だと思ってた」
```

### テクニック05: 「本当は教えたくない」フレーム

商品内容の公開（SEC15）において、一部の項目に「秘匿性」を付与する。

```
✕「骨盤の歪みを整えるストレッチ」
◯「なぜ◯◯を◯◯するだけで△△が変わるのか？」
```

### テクニック06: 逆説的信頼構築

あえて商品の限界や注意点を1つだけ正直に伝える。

例:
- 「ただし、1日で変わるものではありません」
- 「即効性を期待する方には向きません」

完璧すぎる商品は逆に怪しい。1つの「正直な欠点」が全体の信頼性を底上げする。
