# Phase 1: 競合リサーチ

## 概要

コンテンツ作成に入る前に、同テーマの競合商品を分析し、
書くべきネタ・差別化ポイント・言語化のヒントを収集する。

**核心原則: 売れてる商品から逆引きリサーチ**
「何を作るか」ではなく「何が売れているか」からスタートする。
売れている商品には必ず売れている理由がある。その構造を解剖する。

**手法**: 逆引きリサーチ + アカウント観察法 + 顧客リサーチ・競合リサーチ

---

## 実行手順

### STEP 1: プラットフォーム別リサーチ

同テーマで**実際に売れている**有料コンテンツを特定する。
- クイックモード: **3商品**
- 詳細モード: **5商品**

**対象プラットフォーム: note.com のみ**

**調査方法:**

```
■ note (note.com)
検索: "{テーマ} 有料" / "{テーマ} ノウハウ"
確認: スキ数、コメント数、販売実績表示
注目: 無料部分の構成・内容、タイトルの付け方、有料/無料の境界の切り方
分析対象: 有料記事の無料公開部分（セールスレター相当）を読み込む
売れ筋の見極め: スキ数100以上 or コメント10以上 or 販売実績表示あり
```

**選定基準（優先度順）:**
1. 実際に売れている（レビュー・販売数あり）
2. 価格帯が自分の想定と近い（±50%以内）
3. ターゲット層が重なる
4. 最近1年以内にリリースされた
5. 自分の商品のベンチマークにしたい

**⚡ クイックモードの高速化ルール:**
- Web検索は最大3回まで（note検索2〜3回）
- 各商品ページの閲覧は無料公開部分のみ（深追いしない）
- 検索結果のスニペットから読み取れる情報は、ページを開かずに活用してよい

**📝 詳細モードの検索ルール:**
- Web検索は最大7回まで
- 各商品ページの閲覧は無料公開部分のみ（深追いしない）
- 検索結果のスニペットから読み取れる情報は、ページを開かずに活用してよい


### STEP 2: 競合商品の構造分析

各競合商品について以下を分析する。

**⚡ クイックモード（10項目）:**
```markdown
### 競合 [番号]: [商品名]

1. 販売者: [@アカウント名]
2. 価格: ○○円
3. 推定販売規模: （スキ数・コメント数から推定）
4. メインの訴求（何を約束しているか）:
5. ターゲット像:
6. タイトルの構造:
7. 無料部分の構成（ざっくり）:
8. 強み:
9. 弱み・不足している点:
```

**📝 詳細モード（10項目 + 補足分析）:**
```markdown
### 競合 [番号]: [商品名]

**基本情報（10項目）**
1. 販売者: [@アカウント名]
2. 価格: ○○円
3. 推定販売規模: （スキ数・コメント数から推定）
5. メインの訴求（何を約束しているか）:
6. ターゲット像:
7. タイトルの構造: [テーマワード] + [訴求ワード]
8. 無料部分の構成（何を書いているか、どう引き込んでいるか）:
9. 強み:
10. 弱み・不足している点:

**補足分析**
- コンセプトの型: 目標達成型 / 網羅型 / 問題解決型
- 使われているパワーワード:
- 読者からの不満・要望（レビューから）:
- 自分なら補完できるポイント:
- なぜ売れているか（仮説3つ）:
```

### STEP 3: 売れている要素の逆引き（コンテンツ融合法の準備）

分析した商品から「売れている要素」を抽出する。

**抽出テンプレート:**
```
■ 共通して使われている要素（=市場が求めているもの）
1.
2.
3.
4.
5.

■ 差別化に使われている要素（=独自性のパターン）
1.
2.
3.

■ 価格と内容の相関
- ○○円台: [何が含まれている]
- ○○円台: [何が含まれている]
- ○○円台: [何が含まれている]

■ タイトルパターン
- パターンA: [数字] + [期間] + [成果]
- パターンB: [ターゲット] + [手段] + [結果]
- パターンC: [独自メソッド名] + [訴求]

■ 無料部分の構成パターン
- パターンA: 悩み共感 → 実績 → 内容紹介 → 購入促進
- パターンB: ノウハウ一部公開 → 「続きは有料で」
- パターンC: ストーリー → 解決策の提示 → オファー
```

### STEP 4: 顧客リサーチ（有料コンテンツのレビュー・コメントから抽出）

ターゲットが**今まさに抱えている悩み・欲求**を、noteの有料コンテンツのコメントから読み取る。

**⚡ クイックモードの場合:**
- STEP 2で閲覧した商品ページから読み取れた情報のみで構成する
- 追加のWeb検索は行わない

**📝 詳細モードの場合:**
- 競合商品のレビュー・コメントを個別に確認する

**リサーチ方法:**
```
■ noteの有料記事のコメント・スキコメント
- 「参考になった」「ここが良かった」の共通点
- 「ここが足りない」「もっと知りたい」の共通点
- 購入者が書いている感想の言葉をそのまま記録

```

※ Brain、Threads、Yahoo知恵袋、他SNS等の外部ソースは対象外。note.comの有料コンテンツ周辺の情報のみ使用する

**収集テンプレート:**

⚡ クイックモード:
```
■ ターゲットの悩みTOP5
1. [悩み]
2. ...

■ ターゲットが使う言葉
- [実際に使っている表現をそのまま記録]

■ ターゲットの理想の未来TOP3
1. [こうなりたい]
2. ...
```

📝 詳細モード:
```
■ ターゲットの悩みTOP10
1. [悩み] ← 出現頻度: 高/中/低
2. ...

■ ターゲットが使う言葉
- [実際に使っている表現をそのまま記録]

■ ターゲットの理想の未来TOP5
1. [こうなりたい]
2. ...

■ ターゲットの恐怖TOP5
1. [これだけは避けたい]
2. ...
```

### STEP 5: ネタアイデアの書き出し

競合分析 + 顧客リサーチを踏まえて、自分のコンテンツに入れるべきネタを書き出す。
- クイックモード: **15個**
- 詳細モード: **20個**

**書き出し基準（5カテゴリ）:**

⚡ クイックモード:

| カテゴリ | 説明 | 目標数 |
|---------|------|--------|
| A. 競合共通ネタ | 競合が書いていて自分も書けるネタ（必須要素） | 4個 |
| B. 差別化ネタ | 競合が書いていないが自分は書けるネタ | 4個 |
| C. 弱み補完ネタ | 競合の弱みを補完するネタ | 2個 |
| D. 体験ネタ | 自分の実体験から書ける具体的なノウハウ | 3個 |
| E. 願望ネタ | 読者が「これが知りたかった」と思うネタ | 2個 |

📝 詳細モード:

| カテゴリ | 説明 | 目標数 |

### STEP 6: 差別化ポイントの特定

競合と自分の差別化ポイントを**3つ以上**特定する。

```markdown
## 差別化ポイント

### 1. [差別化要素]
- 競合の現状:
- 自分ならこう差別化:
- 読者にとっての価値:
- これを使ったタイトル案:

### 2. [差別化要素]
...

### 3. [差別化要素]
...
```

**差別化の切り口:**
- 速度: 「○日で」「最短で」
- 簡単さ: 「テンプレ通りに」「コピペで」
- 網羅性→絞り込み: 「○○だけに特化」
- ターゲット特化: 「○○専用」「○○向け」
- メソッド独自性: 独自フレームワーク名
- 結果の具体性: 具体的な数字でゴールを示す
