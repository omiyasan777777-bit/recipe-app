# AUTO-content-system — 最強の有料note構築マシーン

## プロジェクト概要

有料noteコンテンツを企画→設計→執筆→販売まで一気通貫で構築するシステム。
マーケティング・コピーライティングの知識を統合した6フェーズパイプライン。

## ディレクトリ構成

```
AUTO-content-system/
├── CLAUDE.md              ← このファイル（プロジェクト設定）
├── README.md              ← プロジェクト説明
├── post_to_note.py        ← note.com 下書き投稿（Selenium）
├── prompts/               ← 各フェーズのプロンプト定義書
│   ├── 00_master_orchestrator.md  ← 全体統括
│   ├── 00a_self_analysis.md       ← Phase 0: 自己分析・棚卸し
│   ├── 01_competitive_research.md ← Phase 1: 競合リサーチ
│   ├── 02_concept_design.md       ← Phase 2: コンセプト設計
│   ├── 03_product_design.md       ← Phase 3: 商品設計
│   ├── 04_paid_content.md         ← Phase 4: 有料部分作成
│   └── 05_sales_letter.md         ← Phase 5: セールスレター
└── output/                ← 生成物の保存先
```

## 6フェーズパイプライン

| Phase | 名前 | プロンプト | 出力ファイル |
|-------|------|-----------|-------------|
| 0 | 自己分析・棚卸し | 00a_self_analysis.md | 00a_self_analysis.md |
| 1 | 競合リサーチ | 01_competitive_research.md | 01_research.md |
| 2 | コンセプト設計 | 02_concept_design.md | 02_concept.md |
| 3 | 商品設計（+タイトル設計） | 03_product_design.md | 03_product_design.md |
| 4 | 有料部分作成 | 04_paid_content.md | 04_paid_content.md |
| 5 | セールスレター | 05_sales_letter.md | 05_sales_letter.md |

## スキルの使い方

各スキルは対応するフェーズのプロンプトに従って実行する。

- **メニュー1（自己分析）** → Phase 0: モード選択（簡易/詳細）→ 簡易=テンプレ+2〜3問（約1分）/ 詳細=会話形式で13問（約5～10分）（`prompts/00a_self_analysis.md` に従う）
- `/research` → Phase 1のプロンプトに従い、Web検索で競合リサーチを実行（クイック/詳細の2モード）
- `/persona` → Phase 0+2を組み合わせ、ターゲットペルソナを設計
- `/outline` → Phase 3のプロンプトに従い、章構成・カリキュラムを設計
- `/write` → Phase 4のプロンプトに従い、たこすけ式構成法で本文を執筆
- `/proofread` → 全フェーズの品質基準に照らして校正チェック
- `/title` → Phase 3 PART Dのプロンプトに従いタイトルを生成
- 「noteに投稿して」→ `post_to_note.py` で note.com に下書き投稿（Selenium）
  - 使い方: `python post_to_note.py [--dir output2] [--title "タイトル"] [--profile "Profile 1"]`
  - 05_sales_letter.md（無料部分）+ 04_paid_content.md（有料部分）を結合して投稿
  - タイトルは 03_product_design.md から自動取得


## 統合知識ベース

### コンセプト設計の核心

- **たこすけ式構成法**: Problem → Amplify → Solution → Testimony → Offer
- **13要素下書き**: ①テーマ〜⑬提案の構造で各章を設計
- **コンテンツ融合法**: 競合3商品の良い要素を融合 + 独自要素 = オリジナル

### セールスの核心

### SNS運用の核心
- **5つのポスト型**: キャラ推し / 実績アピール / 有益ノウハウ / 論理ロジック / ストーリー
- **連載ストーリー型**: 1投稿=1話、複数投稿=1編、10-50投稿=ローンチストーリー
- **ファン化3フェーズ**: フォロワー → ファン → 信者

## フォルダ自動採番

コンテンツ制作を開始する際（Phase 0 のヒアリング開始時）、以下のルールでワーキングフォルダを決定する。

### 自動採番ルール
1. `output/` フォルダが**存在しない** → `output/` を作成してそこに保存
2. `output/` フォルダが**既に存在する** → 次の枝番フォルダを自動作成
   - 既存フォルダを走査: `output2/`, `output3/`, ... を確認
   - 最大番号+1 のフォルダを作成（例: `output2/` まであれば `output3/` を作成）
3. メニュー「11. 追加コンテンツ」を選択した場合も同じ自動採番ルールで新規フォルダを作成

### フォルダ命名規則
```
output/       ← 初回コンテンツ
output2/      ← 2回目
output3/      ← 3回目
output4/      ← 4回目
...
```

### 動作
- フォルダ作成時にユーザーへ「📁 ワーキングフォルダ: output3/ を作成しました」と通知する
- 各フェーズの出力は現在のワーキングフォルダに保存する
- 追加コンテンツも同じ6フェーズパイプラインで制作する

## セットアップ（初回起動）

ユーザーが「セットアップ」と入力したら、以下のメニューを表示する：

```
============================================================
  AUTO-content-system セットアップ
============================================================

  1. note.com ログイン設定
     → Chromeを起動してnote.comにログイン。
       Cookieが保存され、以降の投稿が自動化されます。

  2. スキップして制作を開始
     → 既にログイン済み or 後でログインする場合。

============================================================
  選択してください（1 or 2）
============================================================
```

- **1を選択** → `python setup_note.py` を実行。Chromeが起動しnote.comログインページが開く。ユーザーがブラウザ上でログインするとCookieが保存され、セットアップ完了。完了後にスキル一覧を表示する。
- **2を選択** → ログインをスキップし、スキル一覧を表示してコンテンツ制作に進む。

セットアップ完了後（またはスキップ後）にメインメニューを表示する：

```
============================================================
  AUTO-content-system メインメニュー
============================================================

  [コンテンツ制作]
  1. 自己分析 ⚡簡易     （Phase 0 / 約1分）
  2. 自己分析 📝詳細     （Phase 0 / 約10〜15分）
  3. 競合リサーチ        （Phase 1）
  4. ペルソナ設計        （Phase 0+2）
  5. 章構成・カリキュラム  （Phase 3）
  6. タイトル生成
  7. 本文執筆            （Phase 4）
  8. 校正チェック
  9. セールスレター       （Phase 5）

  [noteアカウント]
  10. アカウント切り替え / 追加
  11. 追加コンテンツ（新規制作）

  現在のアカウント: （アカウント名を表示）
------------------------------------------------------------
  💡 番号を入力してください！初めての方は 1 or 2 のヒアリングから！
============================================================
```

ユーザーが「メニュー」と入力した場合もこのメインメニューを表示する。

各番号の動作：
- **1** → Phase 0 自己分析・簡易モード（`prompts/00a_self_analysis.md` PART Aに従って実行）
- **2** → Phase 0 自己分析・詳細モード（`prompts/00a_self_analysis.md` PART Bに従って実行）
- **3〜9** → 対応するスキル（`/research`, `/persona`, `/outline`, `/title`, `/write`, `/proofread`, `/salesletter`）を実行
- **10** → アカウント切り替えメニューを表示（後述）
- **11** → 追加コンテンツの制作を開始

## noteアカウント管理

アカウント情報は `accounts.json` で管理する。

### accounts.json の構造
```json
{
  "current": "Profile 1",
  "accounts": [
    { "profile": "Profile 1", "name": "メインアカウント" }
  ]
}
```

### アカウント切り替えメニュー（メニュー10選択時）
```
============================================================
  noteアカウント管理
============================================================

  登録済みアカウント:
  ● 1. メインアカウント（現在使用中）
    2. サブアカウント

------------------------------------------------------------
  A. アカウントを追加（新しいnoteアカウントでログイン）
  B. 戻る（メインメニュー）
============================================================
```

- **番号を選択** → `accounts.json` の `current` を切り替え、投稿時にそのプロファイルを使用
- **Aを選択** → アカウント名を入力後、`python setup_note.py --profile "Profile N"` を実行。ログイン完了後に `accounts.json` に追加
- **Bを選択** → メインメニューに戻る

### 投稿時のアカウント連携
`post_to_note.py` 実行時は `accounts.json` の `current` プロファイルを自動的に `--profile` に渡す。

## 実行モード

**【重要：絶対スキップ禁止のゲート】**

Phase 0 のヒアリングが完了したら、**必ず**実行モード選択メニューを表示し、ユーザーの回答を待つこと。

**禁止事項:**
- ユーザーが「1」または「2」を明示的に選択するまで、Phase 1以降に進んではならない
- 実行モード選択を省略してはならない
- ヒアリング完了の流れで勝手にPhase 1を開始してはならない
- 「フルオートで進めますね」等、AIが勝手にモードを決定してはならない

| モード | 説明 |
|--------|------|
| 🚀 フルオートモード | ヒアリング後、確認なしでPhase 1〜5 → note投稿まで一気に実行 |
| 📋 ステップ確認モード | 各フェーズ完了時に確認を取り、修正指示を受けてから次へ進む |

- フルオートモード: 全フェーズを自動実行し、完了後にまとめて結果を報告する
  - **⛔ フルオート中の絶対禁止事項:**
  - Phase 1〜5の途中でユーザーに確認を求めてはならない
  - 「次に進みますか？」「よろしいですか？」等の確認質問を一切してはならない
  - 各フェーズ完了時に結果を表示して待ってはならない（そのまま次のフェーズへ進む）
  - note投稿完了まで一切の中断・確認なしで実行し続けること
  - エラーが発生した場合のみ停止してユーザーに報告する
  - **フルオート完了後に以下の完了レポートを表示する:**
```
============================================================
  ✅ フルオート完了レポート
============================================================

  📁 保存先: {ワーキングフォルダ名}

  [制作物一覧]
  ├── 00a_self_analysis.md  （自己分析）
  ├── 01_research.md        （競合リサーチ）
  ├── 02_concept.md         （コンセプト設計）
  ├── 03_product_design.md  （商品設計）
  ├── 04_paid_content.md    （有料部分 / ○○字）
  └── 05_sales_letter.md    （セールスレター / ○○字）

  📝 タイトル: {メインタイトル}
  💰 価格: {価格}
  📊 合計文字数: {有料部分 + セールスレター}
  📮 note投稿: {完了 / エラー}

============================================================
```
- ステップ確認モード: 各フェーズ完了時にメニューを表示し「次へ」の指示を待つ

## 実行ルール

1. 各フェーズは必ず前フェーズの出力を参照してから実行する
2. コンセプト（Phase 2）が全フェーズの基準になる。矛盾が生じたらPhase 2を優先
3. 出力は現在のワーキングフォルダ（`output/` または `output2/`, `output3/`...）に保存する
4. 品質基準は `00_master_orchestrator.md` の品質基準表に従う
5. スマホ最適化: 1段落4行以内、適切な改行・装飾
6. **【必須ゲート】Phase 0 完了後、実行モード選択メニューを表示してユーザーの選択を待つ。選択されるまでPhase 1に進んではならない**
7. **【必須ゲート】各STEPの質問はユーザーの回答を待ってから次に進む。先回りして複数ステップを同時に実行しない**
