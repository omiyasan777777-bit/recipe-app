事前に用意するもの

作業開始前に以下をすべて準備してください。

  • Threads アカウント（Instagramアカウントに紐づいているもの）
  • Buffer アカウント（buffer.com で無料作成）
  • Manus アカウント（manus.im で無料作成）
  • PCまたはスマホのブラウザ（Chrome推奨）
注意 Threadsの自動投稿にはInstagramアカウントとの連携が必須です。Instagramにログインできる状態にしておいてください。
3

Buffer アカウント＆API設定

3-1. Bufferアカウントを作成する

  1. buffer.com にアクセスし「Get started for free」をクリック
  2. メールアドレスとパスワードを入力して登録（Googleアカウント連携も可）
  3. 届いた確認メールのリンクをクリックしてアクティベート
  4. ダッシュボードにログインできれば完了

3-2. Buffer APIキーを取得する

  1. Buffer ダッシュボード右上のアバターアイコン →「Account Settings」をクリック
  2. 左メニューの「API Access」を選択
  3. 「Create Token」ボタンをクリック
  4. トークン名を入力（例：manus-threads-bot）して「Generate」
  5. 表示されたトークンを 必ずこの画面でコピーして保存（再表示不可）

コピーしたAPIキーの例

# Buffer API Access Token（例） BUFFER_TOKEN=1/eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx.yyy

このトークンはパスワードと同等です。絶対に他人に教えないでください。

重要 APIキーはページを閉じると二度と表示されません。メモ帳やパスワードマネージャーに今すぐ保存してください。
4

ThreadsをBufferに接続する

  1. Buffer ダッシュボードの左サイドバー「Channels」→「Connect a Channel」
  2. SNS一覧から「Threads」を選択
  3. 「Connect Threads」ボタンをクリックするとMeta/Instagramのログイン画面が開く
  4. Instagramのユーザー名とパスワードを入力してログイン
  5. 「Bufferに〇〇へのアクセスを許可しますか？」→「許可」をクリック
  6. Bufferダッシュボードに戻り、Threadsアイコンが表示されれば接続完了
確認方法 左サイドバーの「Channels」にThreadsのアイコンが緑色で表示されていれば正常に接続されています。

チャンネルIDを確認する（後で使います）

  1. Buffer ダッシュボードで Threads チャンネルをクリック
  2. URLを確認：https://publish.buffer.com/profile/xxxxxxxxxx/queue
  3. この xxxxxxxxxx の部分が チャンネルID です。メモしておく
5

Manusのセットアップ

5-1. Manusアカウント作成

  1. manus.im にアクセスし「Get Started」をクリック
  2. Googleアカウントまたはメールアドレスでサインアップ
  3. ダッシュボードにログインし、初回チュートリアルを完了させる

5-2. Manus に環境変数（秘密情報）を登録する

ManusエージェントがBuffer APIを呼び出せるよう、APIキーとチャンネルIDを登録します。

  1. Manusダッシュボード右上の「Settings（⚙）」アイコンをクリック
  2. 「Secrets / Environment Variables」セクションへ移動
  3. 「Add Secret」ボタンをクリックし、以下の2つを登録する

登録する変数

# 変数名 値 BUFFER_TOKEN = （手順3-2でコピーしたAPIキー） BUFFER_CHANNEL = （手順4でメモしたチャンネルID）
なぜSecretに登録するの？ プロンプトに直接APIキーを書くとログに残る危険があります。Secrets機能を使えば暗号化されて安全に管理できます。
6

自動化ワークフローを構築する

6-1. Manusでエージェントタスクを作成する

Manusの新規タスク画面を開き、以下のプロンプトをそのままコピー＆ペーストしてください。

Manus プロンプトテンプレート（コピー用）

以下の手順を実行してください。 1. 今日のトレンドを参考に、Threads向けの短い投稿文を1件作成する。 - 150文字以内 - 絵文字を1〜3個含める - ハッシュタグを2〜3個末尾に追加する 2. 作成した投稿文を Buffer API で予約投稿する。 - エンドポイント: https://api.bufferapp.com/1/updates/create.json - 認証: Bearer $BUFFER_TOKEN - profile_ids: $BUFFER_CHANNEL - scheduled_at: 今から2時間後のUTC時刻（ISO 8601形式） - text: 手順1で作成した投稿文 3. APIレスポンスを確認し、成功した場合は投稿IDを報告する。 失敗した場合はエラー内容を報告する。

6-2. Buffer APIのリクエスト仕様（参考）

# Buffer API — 投稿作成リクエスト例 POST https://api.bufferapp.com/1/updates/create.json # ヘッダー Authorization: Bearer ${BUFFER_TOKEN} Content-Type: application/x-www-form-urlencoded # ボディ profile_ids[]=${BUFFER_CHANNEL} text=今日も一日がんばろう！🌅 #朝活 #Threads scheduled_at=2026-03-31T09:00:00+0000 now=false

6-3. 定期実行（スケジュール）の設定

毎日自動で動かすには、Manusのスケジュール機能を使います。

  1. 作成したタスクの「…」メニュー →「Schedule」をクリック
  2. 実行頻度を選択（例：毎日 07:00 JST）
  3. 「Save Schedule」で保存
  4. タスク一覧に時計アイコンが表示されれば設定完了
おすすめのスケジュール Threadsのエンゲージメントが高い時間帯は 朝7〜9時 と 夜20〜22時（JST）です。まずは1日1回から始めてみましょう。
7

テスト投稿＆動作確認

手動でテスト実行する

  1. Manusダッシュボードでタスクを開き「Run Now」ボタンをクリック
  2. エージェントのログが流れ始めるのを確認する
  3. ログ末尾に "id": "xxxxxxxxxx", "status": "buffer" と表示されれば成功
  4. Bufferダッシュボード →「Queue」タブで予約された投稿を確認する
  5. Threadsアプリで実際に投稿されたか確認する

エラーが出た場合のチェックリスト

  • BUFFER_TOKEN が正しくコピーされているか（前後にスペースがないか）
  • BUFFER_CHANNEL がThreadsチャンネルのIDになっているか
  • BufferのThreadsチャンネルが「接続済み」になっているか
  • Bufferの無料プランの月10件制限に達していないか
  • scheduled_at の時刻が現在より未来になっているか

運用のコツ＆応用例

投稿クオリティを上げるプロンプト改善

プロンプトに以下を追加するとよりターゲットに刺さる投稿になります。

  • 「ターゲット読者：30代女性、副業・節約に興味あり」
  • 「口調：友達に話しかけるようなフランクな文体」
  • 「過去の投稿との重複を避けること」

複数SNSへの同時投稿

Buffer の profile_ids[] に複数のチャンネルIDを渡すと、ThreadsとX（Twitter）、Instagramに同時投稿できます。

profile_ids[]=threads_channel_id profile_ids[]=twitter_channel_id profile_ids[]=instagram_channel_id

noteやブログの更新を自動告知

ManusにRSSフィードを監視させ、新記事が公開されたら自動でThreadsに告知投稿することも可能です。プロンプトに以下を追加してください。

# noteの最新記事を取得して投稿文を生成する RSSフィード: https://note.com/あなたのユーザーID/rss 未投稿の最新記事タイトルとURLを取得し、 「新しい記事を公開しました！」という紹介文と 一緒にThreadsへ投稿してください。

よくある質問

Bufferの無料プランでは何件まで投稿できますか？
無料プランでは月10件まで予約投稿できます。毎日投稿したい場合は月$6〜の有料プランへのアップグレードをおすすめします。年払いにするとさらに割安になります。
Manusは日本語に対応していますか？
はい、Manusは日本語のプロンプトに対応しています。プロンプトをそのまま日本語で書いても問題なく動作します。
投稿が重複してしまいます。どうすればいいですか？
Manusのプロンプトに「過去7日間に投稿した内容と重複しないようにすること」と明記してください。また、Bufferの「Sent」タブに蓄積された投稿履歴をManusに読み込ませる仕組みを追加するとより効果的です。
Buffer APIのレート制限はありますか？
Buffer APIには1分あたり約60リクエストのレート制限があります。大量の投稿を一度に作成しようとするとエラーになる場合があります。1リクエストずつ間隔を置いて処理するようManusのプロンプトに指示を追加してください。
Threadsのアカウントが凍結されないか心配です。
Buffer はMeta公認のAPIパートナーであるため、Buffer経由の投稿は規約違反になりません。ただし、1日に大量の投稿（目安：10件以上）を行うとスパム判定されるリスクがあります。1日1〜3件程度の頻度に抑えることをおすすめします。
ManusとBufferの料金が気になります。無料で運用できますか？
Manusの無料プランとBufferの無料プランを組み合わせれば、月10件までは完全無料で運用できます。毎日投稿したい場合はBuffer有料プラン（月$6〜）への移行を検討してください。
